# Week 05 - Exercise 03 (Distributed Architecture Design)

Topic: Distributed architecture style for Online Food Delivery.

This exercise focuses on migration design from monolith to microservices.

## Required outcomes

1. Design monolith to microservices migration.
2. Identify bounded contexts.
3. Describe sync and async communication between services.
4. Propose event messaging model with Kafka or RabbitMQ.

## Deliverables

1. Service map: docs/service-map.md
2. Context map: docs/context-map.md
3. Communication diagram: docs/communication-diagram.md
4. Event messaging design: docs/event-messaging-design.md
5. Migration plan: docs/migration-plan.md

## Mapping to implementation exercises

1. Exercise 04 implements the monolith version with 3 fullstack functions.
2. Exercise 05 implements the microservices version with the same 3 fullstack functions.
